오늘은 깃과 깃허브, 그리고 커밋에 관한 전반적인 내용들을 배웠다.
Git은 프로젝트의 버전 관리를 위한 도구로 파일의 내용을 추가하거나 삭제하는 등 수정할때마다 히스토리 내역에 날짜와 시간과 함께 추가, 삭제 내역등이 ㅈ장된다.
GitHub는 깃을 확장한 개념으로 Git 원격 저장소 + Git 으로 할 수 있는 커뮤니티 기능 서비스이다.
이 원격 저장소 기능을 이용해서 여러 개발자들이 하나의 프로젝트를 관리할 수 있다.
이런 GitHub와 같은 프로젝트 관리 기능을 제공하는 도구로는 Gitlab과 bibucket 가 있다.

sourctree는 git을 쉽게 사용할 수 있게 돕는 도구로 오늘은 이 sourctree를 이용하여 git 프로젝트를 생성하고 각 기록들을 commit하는 방법과 pull을 통해 원격repo(repo는 git으로 관리되는 프로젝트를 말한다)에 로컬repo의 수정내역과 파일을 저장하거나, 반대로 push를 통해 원격repo의 수정내역과 파일을 로컬repo에 저장하는 방법, clone를 통해 빈 초기 폴더에 원격repo의 수정내역과 파일을 저장하는 방법을 배웠다.
