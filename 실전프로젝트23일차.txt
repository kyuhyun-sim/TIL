실시간 알림을 구현하기 위해 고려해본 기술들
polling, websocket, sse 이렇게 3가지가 후보로 선택됨.
이중 sse 방식을 선택하기로 함
이유는 먼저 3가지 방식의 특징을 알아 볼 필요가 있음.

polling 방식의 경우 클라이언트가 일정 주기로 서버에 업데이트 요청을 보내어 지속적으로 http 요청이 발생하게 됨.
리소스 낭비가 심해짐

웹소켓 방식의 경우 실시간 양방향 통신에 사용되는 방식으로 서버와 브라우저가 TCP 라인을 통해 연결되어 지속적으로 실시간 데이터를 주고 받을 수 있도록 해줌.
연결 지향의 양방향 전이중 통신이 가느하기에 채팅, 게임 등에 사용됨.

SSE 방식의 경우 서버 -> 클라이언트 방향으로만 신호가 가는 단방향 통신 채널로, SSE는 클라이언트과 폴링방식과 같이 주기적으로 http 요청을 보내줄 필요가 없음.
http 연결을 통해 서버에서 클라이언트로 데이터를 보내줌.

각 방식을 확인 끝에 SSE 방식을 선택하기로 함.
프로젝트에서 구현할 실시간 알림은 오로지 서버에서 클라이언트측으로 보내주는 신호 밖에 없기 때문에 클라이언트에서 서버로 리소스가 발생하는 폴링방식이나 양방향 통신인 웹소켓 보다는 서버에서 클라이언트로 단방향 통신만 하는 SSE 방식이 리소스를 가장 덜 잡아 먹는 방식으로 보임.
